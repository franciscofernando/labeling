Introducción

Labeling intenta ser un MV*, simple y flexible, rescata las buenas ideas de sus pares

Atributo Name

Lo principal en labeling es el atributo "name", este le da una identidad al elemento y lo transforma en un entorno.
Ejemplo: <div name="elemento" > </div>

Modelo

El modelo es espacio donde se guardan los atributos y metodos del entorno, se inicializa con el nombre del entorno y un objeto con los datos.
Ejemplo: model.export('elemento', { atributo : 'Texto' });

Controlador

El controlador es una función que agrupa los atributos y metodos para el entorno.
Ejemplo: controller.export('elemento', function(){ console.log(this.atributo); });

Atributos

Los atributos de un controlador son los datos del modelo, estos se rescatan con "this".
Ejemplo: controller.export('elemento', function(){ alert('Este es el valor de atributo'+this.atributo); });

Metodos

Los metodos son funciones que son ejucutadas en el controlador, estas pueden de ser tipo evento o lineales.
Ejemplo: controller.export('elemento', function(){ this.metodo = function(texto){ this.atributo = 'Texto cambiado'; } });

Bind(Lazo)

Dentro de un elemento(que contenga solo texto) se puede poner etiquetas bind(Lazos) de esta manera: {{nombreDelAtributo}}

Binding: {{#bind}} = {{bind}}

Herencia

Si hay un entorno dentro de otro entorno, este hereda los datos de su modelo solo si el entorno hijo no tiene datos con el mismo nombre:

  • Variable heredada: {{#bind}} = {{bind}}
  • Variable propia: {{#propio}} = {{propio}}

Label(Etiqueta)

No solo el entorno puede ser identificado sino tambien los elementos con el atributo label, esto sirve para usar el elemento de una manera mas facil en el controlador.
Ejemplo: <button label="boton">BOTON</button>

Llamar al elemento etiquetado

Para llamar a la/las etiquetas del entorno se tiene que recibir el primer parametro del controlador.
Ejemplo: controller.export('elemento', function(labels){ console.log(labels.boton); });

Eventos

Como labeling funciona con jQuery, puedes usar los eventos de jquery para gatillar acciones.
Ejemplo: controller.export('elemento', function(labels){ labels.boton.click(function(){ alert('Haz presionado a boton'); }); })

Modificación de atributos

Si hay un lazo con un atributo, este cambia automaticamente si este cambia en el modelo.

Modificacion de bind con boton

{{#contador}} = {{contador}}

Modificacion de bind con input

{{#texto}} = {{texto}}

Router

El router es un objeto que vigila la url, y ejecuta diferentes eventos relacionados con esta.
Ejemplo: router.export('/url/:variable', function(data){ alert('Ha cambiado la url y la variable es: '+data.variable); });

change

El metodo principal de router, se usa para cambiar la url.
Ejemplo: router.change('/url/admin'); //Usado con la explicacion de router devolvera un alert con: "Ha cambiado la url y la variable es: admin".

Router include

Es un comentario HTML con el formato: <!-- router NOMBRE URL:variable -->(:variable es opcional y sera explicado en la seccion de show).
Ejemplo: <!-- router usuario /user/:variable -->

show

Este metodo imprime el html de la url de "Router include", como parametro opcional esta dar un objeto cuyos atributos se reemplazaran con los nombres de la url.
Ejemplo: router.show('NOMBRE', {variable: 1}) //buscara la ruta http /user/1

hidden

Este metodo se utiliza para borrar el contenido agregado por router.show.
Ejemplo: router.hidden('NOMBRE'); //borrara todo lo agregado al ejecutar router.show('NOMBRE')

Se incrusto el comentario HTML <!-- router routerload routerLoad.html?nombre=:name -->

Show y change

router.export('/section/:name', function(data){ router.show('routerload', data); });
router.change('/section/home');

Hidden y change

router.export('/', function(data){ router.hidden('routerload'); });
router.change('/');

Include

Se puede incluir el contenido de una url en el lugar donde esta el include, para esto hay que incrustar un comentario HTML, la url puede estar con bind y el include dentro de un entorno.
Ejemplo: <!-- include getLoad.html -->
